## 🔄 استعمال شدہ API اینڈ پوائنٹس

### GET پروفائل
```
GET /api/auth/vendor/profile
Headers: Authorization: Bearer <token>
```

**جواب:**
```json
{
  "id": 1,
  "email": "vendor@example.com",
  "phone": "03001234567",
  "businessName": "میری دکان",
  "ownerFirstName": "احمد",
  "ownerLastName": "علی",
  "businessType": "ریستوران",
  "shopAddress": "123 مین سٹریٹ",
  "city": "خرطوم",
  "ntnNumber": "12345",
  "status": "active",
  "rating": 4.5,
  "reviewsCount": 120,
  "totalProducts": 25,
  "totalOrders": 340,
  "totalEarnings": 125000.50,
  "createdAt": "2024-01-15T10:30:00Z",
  "userType": "vendor"
}
```

### PUT پروفائل اپ ڈیٹ
```
PUT /api/auth/vendor/profile
Headers: Authorization: Bearer <token>
Content-Type: application/json
```

**درخواست کی باڈی:**
```json
{
  "ownerFirstName": "احمد",
  "ownerLastName": "علی",
  "phone": "03001234567",
  "businessType": "ریستوران",
  "shopAddress": "123 مین سٹریٹ، خرطوم",
  "city": "خرطوم"
}
```

**جواب:**
```json
{
  "success": true,
  "message": "وینڈر پروفائل کامیابی سے اپ ڈیٹ ہو گیا"
}
```

## 🔒 حفاظتی خصوصیات

1. **محفوظ اسٹوریج**
   - تمام auth tokens `flutter_secure_storage` میں محفوظ
   - ڈیوائس پر خودکار انکرپشن

2. **خودکار ٹوکن Refresh**
   - Dio interceptor تمام درخواستوں میں ٹوکن شامل کرتا ہے
   - 401 errors خودکار طور پر auth ڈیٹا صاف کرتے ہیں

3. **لاگ آؤٹ کی حفاظت**
   - لاگ آؤٹ سے پہلے تصدیقی ڈائیلاگ
   - تمام سیشن ڈیٹا صاف کرتا ہے
   - لاگ آؤٹ کے بعد واپس نیویگیشن کو روکتا ہے

## 🔧 بیک اینڈ کی ضروریات

یقینی بنائیں کہ آپ کے بیک اینڈ میں یہ endpoints ہیں:

1. `GET /api/auth/vendor/profile` - وینڈر پروفائل واپس کرتا ہے
2. `PUT /api/auth/vendor/profile` - وینڈر پروفائل اپ ڈیٹ کرتا ہے
3. دونوں کو Authorization header میں JWT token کی ضرورت ہے
4. بیک اینڈ فون نمبر کے فارمیٹ کی توثیق کرتا ہے (03XXXXXXXXX)
